How fast is EV adoption growing across Ontario?
Caches 17 quarterly CSVs of EV registrations. Runs time series with window functions. Province-wide EVs grew 3.4√ó from Q1 2022 to Q4 2025 (73k ‚Üí 249k). Toronto-area FSAs show the fastest growth rates.
Source: EV Registrations by FSA
Under the hood

Follows: EV adoption
Which postal codes saw the biggest EV explosion?
Your 17 quarterly CSVs are already cached. Compares Q1 2022 to Q4 2025 per FSA. M9W (Etobicoke) jumped from 139 to 1,465 EVs ‚Äî a 10.5x increase. Province-wide: 73k to 248k, or 3.4x.
Source: EV Registrations by FSA (already cached)
Under the hood
Skips search ‚Äî EV data already in DuckDB
query_cached
SELECT q1.FSA, q1.TotalEV AS ev_2022, q4."Total EV" AS ev_2025, ROUND(q4."Total EV" * 1.0 / q1.TotalEV, 1) AS growth FROM ev_q1_2022 q1 JOIN ev_q4_2025 q4 ON q1.FSA = q4.FSA WHERE q1.TotalEV > 50 ORDER BY growth DESC

Did ambulance response times get worse after COVID?
Re-uses cached ambulance response data. Compares 2019 to 2023 by coverage area. Every single area got slower. Thunder Bay went from 7 min 22 sec to 10 min 1 sec. Algoma: 8:57 to 11:46.
Source: Ambulance Response Times (already cached)

Uses 439k cached readings (1976-2024). Averages total phosphorus by year. Dropped from 0.044 mg/L in 1976 to 0.022 in the 1990s. Rose to 0.031 in the 2000s, then fell back to 0.024 by 2023.

How it works

Each question triggers a pipeline of tool calls.

1
Search
Finds relevant datasets across 25 ministries. Keyword, topic, location, or related-dataset lookup.
2
Retrieve
Downloads CSVs, XLSX, shapefiles, GeoJSON. Parses and caches locally in DuckDB.
3
Analyze
Runs SQL against cached data. Aggregations, joins, window functions ‚Äî results come back as insight.

Get started

Works with any MCP-compatible client. No API key required.

Add to claude_desktop_config.json

{
  "mcpServers": {
    "ontario-data": {
      "command": "uvx",
      "args": ["ontario-data-mcp"]
    }
  }
}

Add via Claude Code CLI

claude mcp add ontario-data -- uvx ontario-data-mcp

Add to .vscode/mcp.json

{
  "mcpServers": {
    "ontario-data": {
      "command": "uvx",
      "args": ["ontario-data-mcp"]
    }
  }
}

Run directly with uvx

uvx ontario-data-mcp
Or clone from source: git clone https://github.com/sprine/ontario-data-mcp && cd ontario-data-mcp && uv sync
